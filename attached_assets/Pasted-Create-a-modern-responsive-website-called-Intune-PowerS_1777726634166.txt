4. Detection script skeleton
Generate a strong detection script skeleton for Intune proactive remediations.

Example detection behavior:
- Check if registry path exists
- Check if registry value exists
- Read current value
- Compare current value to expected value
- If compliant, output a clear message and exit 0
- If non-compliant, output a clear message and exit 1
- If error occurs, output error details and exit 1

Include comments explaining:
- Exit 0 means compliant / no remediation needed
- Exit 1 means non-compliant / remediation should run
- Write-Output is visible in Intune logs
- Avoid changing anything in detection scripts

Example generated detection script skeleton:

```powershell
<#
.SYNOPSIS
Detects whether a registry value matches the expected configuration.

.DESCRIPTION
Designed for Microsoft Intune proactive remediations.
This detection script checks a registry value and exits:
0 = compliant
1 = non-compliant, remediation required

.NOTES
Test before production deployment.
#>

$RegistryPath = "HKLM:\SOFTWARE\Contoso\Settings"
$ValueName = "Enabled"
$ExpectedValue = 1

try {
    if (-not (Test-Path $RegistryPath)) {
        Write-Output "Registry path not found: $RegistryPath"
        exit 1
    }

    $CurrentValue = Get-ItemPropertyValue -Path $RegistryPath -Name $ValueName -ErrorAction Stop

    if ($CurrentValue -eq $ExpectedValue) {
        Write-Output "Compliant: $ValueName is set to $ExpectedValue"
        exit 0
    }
    else {
        Write-Output "Non-compliant: $ValueName is $CurrentValue, expected $ExpectedValue"
        exit 1
    }
}
catch {
    Write-Output "Detection failed or value missing: $($_.Exception.Message)"
    exit 1
}
